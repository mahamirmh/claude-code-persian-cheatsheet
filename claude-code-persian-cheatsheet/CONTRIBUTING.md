# راهنمای مشارکت

از مشارکت برای بهتر شدن این راهنمای فارسی استقبال می‌شود.

## چه چیزهایی قابل مشارکت است؟

- اصلاح دستورهای قدیمی یا اشتباه
- اضافه‌کردن workflowهای جدید
- اضافه‌کردن پرامپت‌های کاربردی
- بهترکردن توضیحات فارسی
- اضافه‌کردن مثال‌های واقعی
- اصلاح لینک‌های منابع رسمی

## قوانین مشارکت

1. دستورها را تا حد ممکن با مستندات رسمی Claude Code بررسی کن.
2. ادعای قطعی درباره دستورهای نسخه‌های آینده نده.
3. اگر دستوری فقط در بعضی نسخه‌ها فعال است، در توضیح بنویس.
4. اطلاعات شخصی، API Key، Token یا داده خصوصی وارد نکن.
5. متن فارسی را ساده و کاربردی نگه دار.

## مراحل پیشنهادی

```bash
git checkout -b fix-or-feature-name
# edit files
git add .
git commit -m "Update Claude Code commands"
git push origin fix-or-feature-name
```

بعد Pull Request بساز.
