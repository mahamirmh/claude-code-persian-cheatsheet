# پرامپت‌های آماده کدنویسی

## تحلیل پروژه

```text
Analyze this codebase and explain:
1. project structure
2. main entry points
3. build/run commands
4. risky areas
5. what I should read first
Do not edit files yet.
```

## دیباگ

```text
Find the root cause of this bug. Do not make changes until you explain:
1. likely cause
2. files involved
3. safest fix
4. tests needed
Bug:
...
```

## توسعه قابلیت جدید

```text
Implement this feature with the smallest safe change. Before editing, create a plan. After editing, run tests or explain why tests cannot run.
Feature:
...
```

## بررسی diff

```text
Review my current git diff for:
1. correctness bugs
2. security issues
3. broken edge cases
4. unnecessary complexity
Do not edit files.
```

## تست

```text
Generate unit tests for the changed files. Focus on edge cases, failure paths, and regression risks.
```
